# C2605 数据源优化方案

## 🎯 优化目标
将C2605玉米期货数据获取逻辑重构为：
1. **主要数据源**: MCP服务器 `dashscope-websearch` 
2. **移除Mock数据**: 失败时直接报错，不再使用模拟数据
3. **简化架构**: 专注于单一高质量数据源

## 📋 当前问题
- 当前使用AKShare历史数据（有1天延迟）
- Mock数据兜底可能导致错误信息传递
- 海外服务器访问限制导致实时数据获取困难

## ✨ 优化方案

### 主要数据源：dashscope-websearch MCP服务器
- **优势**: 
  - AI优化搜索，无广告干扰
  - 支持中文搜索和时间过滤
  - 包含AI生成的直接答案
  - 专为Agent优化，结果精准
- **查询策略**:
  - 查询: "大连商品交易所 玉米期货 C2605 最新价格 实时行情"
  - 时间过滤: 最近24小时
  - 主题过滤: finance

### 错误处理策略
- **移除Mock数据**: 不再生成模拟价格
- **严格错误处理**: 
  - 数据获取失败 → 直接抛出异常
  - 验证失败 → 抛出异常  
  - 网络错误 → 抛出异常
- **日志记录**: 详细记录失败原因

### 验证机制
- **内部验证**: dashscope-websearch自身包含AI验证
- **格式验证**: 确保返回价格数据格式正确
- **合理性检查**: 价格范围合理性验证（如2000-3000元/吨）

## 📝 配置要求

### MCP服务器配置
确保系统已配置dashscope-websearch MCP服务器：
```json
{
  "name": "dashscope-websearch",
  "type": "mcp",
  "config": {
    "endpoint": "dashscope-websearch",
    "apiKey": "sk-xxx"
  }
}
```

### 权限要求
- 需要访问MCP服务器的权限
- 需要网络访问权限
