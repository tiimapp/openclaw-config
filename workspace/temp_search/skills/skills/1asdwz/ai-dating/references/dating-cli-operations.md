# Dating CLI Detailed Operations

This document is for AI Agents using the `dating` skill.  
It explains `dating-cli` command usage, input fields, and output fields.

## 1. Basic Conventions

- CLI name: `dating-cli`
- Local config file: `~/.dating-cli/config.json`
- Output format: JSON by default, suitable for machine parsing

## 2. Common Rules

### 2.1 Parameter Input Rules

- Use `--help` on each command to see supported options.
- Most commands return a standard JSON envelope.

### 2.2 Error Output Structure

```json
{
  "ok": false,
  "error": {
    "message": "...",
    "status": 500,
    "payload": {}
  }
}
```

## 3. Success Output Structure

Most successful commands return:

```json
{
  "ok": true,
  "response": {
    "code": 200,
    "message": "Success",
    "data": {}
  }
}
```

Common extra top-level fields:

| Field | Type | Description |
|---|---|---|
| ok | Boolean | Whether the command succeeded |
| savedTo | String | Config file path where updates were saved |
| configPath | String | Local config path (`config` commands) |
| config | Object | Local config snapshot (`config` commands) |
| taskState | Object | Latest task snapshot (`task` commands) |

## 4. Config Commands

### 4.1 `dating-cli config show`

Purpose: Show current local config without calling backend APIs.

Input fields:

| Field | Type | Required | Description |
|---|---|---|---|
| (none) | - | - | No arguments |

Output fields:

| Field | Type | Description |
|---|---|---|
| ok | Boolean | Always `true` |
| configPath | String | Absolute local config file path |
| config.token | String | JWT token (may be absent) |
| config.tokenHead | String | Token prefix (may be absent) |
| config.memberId | Long | Current logged-in memberId (may be absent) |
| config.username | String | Current username (may be absent) |
| config.updatedAt | String | Last update time (ISO datetime) |

## 7. Task Commands

### 7.1 `dating-cli task create --task-name <name> [criteria options]`

Mapped API: `POST /match-tasks`

Input fields (base):

| Field | Type | Required | Description |
|---|---|---|---|
| --task-name | String | Yes | Task name |

Input fields (criteria):

| Field | Type | Required | Description |
|---|---|---|---|
| --preferred-gender-filter | String(JSON object) | No | Gender filter, e.g. `{"eq":"female"}` |
| --preferred-height-filter | String(JSON object) | No | Height filter, e.g. `{"gte":165,"lte":180}` |
| --preferred-income-filter | String(JSON object) | No | Income filter |
| --preferred-city-filter | String(JSON object) | No | City filter |
| --preferred-nationality-filter | String(JSON object) | No | Nationality filter |
| --preferred-education-filter | String(JSON object) | No | Education filter |
| --preferred-occupation-filter | String(JSON object) | No | Occupation filter |
| --preferred-education-stage | String | No | Education keyword |
| --preferred-occupation-keyword | String | No | Occupation keyword |
| --preferred-hobby-text | String | No | Preferred hobby text |
| --preferred-character-text | String | No | Preferred personality text |
| --preferred-ability-text | String | No | Preferred ability text |
| --hobby-embedding-min-score | Number | No | Min hobby embedding score |
| --character-embedding-min-score | Number | No | Min personality embedding score |
| --ability-embedding-min-score | Number | No | Min ability embedding score |
| --preferred-contact-channel | String | No | Preferred channel: `phone/telegram/wechat/signal_chat/line/snapchat/instagram/facebook` |

Output fields (top-level):

| Field | Type | Description |
|---|---|---|
| ok | Boolean | Command success |
| savedTo | String | Local config save path |
| taskState.taskId | Long | Latest task ID |
| taskState.taskName | String | Latest task name |
| taskState.status | String | Latest task status (`active/stopped`) |
| taskState.matchStatus | String | Latest match status |
| taskState.resultVersion | Integer | Latest task result version |
| taskState.operation | String | Operation name (for example `task.create`) |
| taskState.updatedAt | String | Snapshot write time |

Output fields (`response.data`):

| Field | Type | Description |
|---|---|---|
| taskId | Long | Task ID |
| taskName | String | Task name |
| status | String | Lifecycle status (`active/stopped`) |
| matchStatus | String | Match status (`UNMATCHED/MATCHED/CONTACT_REVEALED/STOPPED`) |
| resultVersion | Long | Result version |
| finalContactMemberId | Long | Final contacted memberId (may be null) |
| lastWatchAt | DateTime | Latest check/watch time |
| lastMatchAt | DateTime | Latest match time |
| matchedAt | DateTime | First matched time |
| contactRevealedAt | DateTime | Contact reveal time |
| createdAt | DateTime | Creation time |
| updatedAt | DateTime | Update time |
| criteria | Object | Task criteria snapshot |

## 10. Recommended CLI Loop

```bash
# 1) Register or login
dating-cli register --username <preferredName>
# or
dating-cli login --username <u> --password <p>

# 2) Update profile
dating-cli profile update --gender male --city Shanghai --character-text "<text>" --hobby-text "<text>" --ability-text "<text>"

# 3) Create task
dating-cli task create --task-name "<name>" --preferred-gender-filter '{"eq":"female"}' --preferred-height-filter '{"gte":165}' --preferred-city-filter '{"eq":"Shanghai"}'

# 4) Check match results (repeat when NO_RESULT_RETRY_NOW)
dating-cli check <taskId> 

# 5) Reveal contact after match
dating-cli reveal-contact <matchId>

# 6) Submit review after communication
dating-cli review <matchId> --rating 5 --comment "Smooth communication"
```

If criteria need to be adjusted and matching should continue:

```bash
dating-cli task update <taskId> --task-name "<name>" --preferred-gender-filter '{"eq":"female"}' --preferred-city-filter '{"eq":"Hangzhou"}'
```
