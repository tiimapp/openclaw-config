# Dashboard Builder Tips

## 最佳实践

1. **先规划再创建** — 确定看板目标和受众，再选择组件
2. **少即是多** — 一个看板建议不超过6-8个组件，避免信息过载
3. **KPI优先** — 把最重要的指标放在最上面，用大数字卡片展示
4. **配色统一** — 选择一个主题并保持一致，避免花哨
5. **响应式布局** — 使用grid布局让看板在不同屏幕上自适应

## 组件选择指南

| 数据类型     | 推荐组件    | 适用场景        |
|-------------|------------|----------------|
| 单一数值     | number     | KPI、核心指标    |
| 趋势变化     | line       | 时间序列数据     |
| 分类对比     | bar        | 类目间比较       |
| 占比分布     | pie        | 市场份额、构成比  |
| 进度跟踪     | progress   | 目标完成度       |
| 明细数据     | table      | 详细记录列表     |

## 主题推荐

- **light** — 适合打印和投影
- **dark** — 适合大屏展示和监控
- **corporate** — 适合商务汇报
- **startup** — 适合产品演示

## 常见问题

- 导出的HTML是自包含的，无需联网即可打开
- SVG图表会自动根据数据调整比例
- 数字卡片支持显示变化趋势（涨/跌箭头）
