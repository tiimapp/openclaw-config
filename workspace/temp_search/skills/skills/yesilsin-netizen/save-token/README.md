# Save Token Skill

## 📁 文件结构

```
save-token/
├── SKILL.md           # Skill 主文件（ClawHub 上传必需）
├── scripts/
│   └── dedupe.js      # 去重脚本（可选，用于实际执行）
└── README.md          # 本说明文件
```

## 🚀 在 ClawHub 上发布步骤

### 方法一：仅上传 SKILL.md（推荐新手）

1. 访问 ClawHub
2. 创建新 Skill
3. 上传 `SKILL.md` 文件
4. 填写 Skill 信息：
   - Name: `save-token`
   - Description: `Token 节省器 - 自动去重上下文中的重复内容`
   - Emoji: `💰`

### 方法二：上传完整文件夹

1. 将整个 `save-token` 文件夹打包为 zip
2. 在 ClawHub 上传 zip 文件
3. 系统会自动识别 `SKILL.md` 和 `scripts/` 目录

## 🎯 Skill 核心功能

### 自动触发机制

此 Skill 设计为在每次任务执行前自动调用：

```
用户请求 → save-token 去重 → 主任务执行
```

### 去重逻辑

1. **扫描上下文**：识别当前对话历史中的所有文本块
2. **检测重复**：找出完全相同的大段文本（≥100字符）
3. **去重处理**：删除重复实例，只保留最早出现的版本
4. **报告结果**：显示节省的 Token 数量

### 去重规则

| 规则 | 说明 |
|------|------|
| 最小长度 | 仅去重 ≥100 字符的文本块 |
| 完全匹配 | 必须字符级完全相同 |
| 保留首次 | 保留最早出现的版本 |
| 保持结构 | 维持消息顺序和角色 |

## 📊 预期效果

| 场景 | 节省 Token |
|------|-----------|
| 文件被多次读取 | 30-50% |
| 长对话中的重复错误信息 | 20-40% |
| 相似问题的多次回答 | 15-30% |

## ⚠️ 注意事项

1. **调用时机**：必须在主任务开始前调用
2. **系统消息**：不会去重 system 角色的消息
3. **数据安全**：仅删除重复内容，独特内容完全保留
