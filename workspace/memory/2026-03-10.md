### 问题报告
**时间:** 2026-03-10 12:29
**用户反馈:** "c2605 的数据不对，排查一下"

### 问题根因
监控脚本 `c2605_monitor.py` 使用的是 `ak.futures_zh_daily_sina()` 获取历史日线数据，然后取 `iloc[-1]` (最后一条记录)，这实际上是**昨天的收盘价**，不是实时行情！

**症状:**
- 报告价格：¥2395.00 (2026-03-09 收盘价)
- 实际时间：2026-03-10 12:00+ (今日交易时段)
- 数据日期：2026-03-09 ❌

### 修复方案

#### 方案 1: 新浪财经实时接口 (优先)
```python
url = "https://hq.sinajs.cn/list=fu_C2605"
# 返回实时行情：open, prev_close, current_price, high, low, volume...
```

**特点:**
- 实时数据 (毫秒级延迟)
- 3 次重试机制
- 超时时间：10 秒

#### 方案 2: AKShare 历史数据 (备选)
```python
ak.futures_zh_daily_sina(symbol="C2605")
# 添加日期验证
if last_date != today:
    print("⚠️ STALE DATA")
```

**特点:**
- 历史日线数据
- 添加日期验证
- 标记为"数据延迟"
